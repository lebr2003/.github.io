Start des Programms: Die Datei wird automatisch aus der Cloud importiert. Der Pfad in der Cloud lautet /Apotheke/Arzneimittel/Arzneimittel_Inventar.txt bzw. /Apotheke/Medizinprodukte/Medizinprodukte_Inventar.txt

Importieren: Ein externer Import (außerhalb der Cloud) wird durchgeführt. Es erscheint eine Warnung. Durch das Importieren außerhalb der Cloud wird sowohl die derzeitige Tabelle im Programm, als auch die Datei in der Cloud überschrieben. Über den Versionsverlauf ist die Wiederherstellung möglich, Account benötigt aber die Freigabe. /Apotheke/Bibliothek.xlsx.

Refresh: Mit Refresh werden alle Einträge, die denselben Barcode und dasselbe Ablaufdatum haben, zusammengeführt. Die Menge wird addiert. Die Einträge werden nach Ablaufdatum sortiert. Zusätzlich werden alle Einträge, die bereits abgelaufen sind oder im nächsten Monat ablaufen, rot markiert. Einträge, die fälschlicherweise eine negative Menge haben, werden gelb markiert und nach oben sortiert.

Ordnen nach Wirkstoff/Bezeichnung: Die Einträge werden alphabetisch nach dem Wirkstoff bzw. der Bezeichnung geordnet. (Abgelaufene Einträge werden nur mit Refresh rot gefärbt.)

Menge: Ein neues Fenster wird angezeigt. Die Einträge der Tabelle werden nach dem Barcode zusammengefasst, dabei werden die Mengen addiert. Das Ablaufdatum wird hier ignoriert. Dadurch sieht man die Gesamtanzahl der einzelnen Produkte.

  • Vergleichen: Die Bibliothek wird automatisch aus der Cloud importiert. Der Pfad in der Cloud lautet /Apotheke/Bibliothek.xlsx. Die Bibliothek enthält u.a. auch die Mindestlagermengen. Das Inventar wird mit der Mindestlagermenge verglichen. In Spalte 2 steht die Inventar-Menge, in Spalte 3 steht die Mindestlagermenge. Ist die Menge eines Produktes im Inventar kleiner oder gleich der Mindestlagermenge, wird die Zeile orange markiert. Fehlt ein Produkt im Inventar, so wird es auch orange markiert, wobei die 3. Spalte dabei leer bleibt.
  • untere Tabelle: Hier werden Produkte angezeigt und orange gefärbt, die sich im Inventar befinden, aber nicht in der Bibliothek stehen.
  • Schließen (x): Das Fenster mit der gerade angezeigten Vergleichstabelle wird geschlossen (und nicht gespeichert). Ein erneutes Aufrufen ist jederzeit möglich.

Speichern: Die Tabelle wird in die Cloud hochgeladen. Die Datei in der Cloud wird überschrieben. Der Pfad in der Cloud lautet /Apotheke/Arzneimittel/Arzneimittel_Inventar.txt bzw. /Apotheke/Medizinprodukte/Medizinprodukte_Inventar.txt. Vor dem Schließen oder Neuladen des Programms kommt immer (außer Safari auf IOS) eine Standardmeldung vom Browser, um darauf hinzuweisen, die Tabelle zu speichern.

Extern Speichern: Die Tabelle wird als .txt exportiert und der Dateiname mit dem heutigen Datum und Uhrzeit versehen. Diese Funktion ist vor allem notwendig, wenn es Fehler mit der API gibt.

On/Off-Schalter: On (blau): Jedes Tabellenfeld kann per Mausklick bearbeitet werden. Alle Buttons und Automatismen sind dabei deaktiviert. Kein Tabellenfeld darf komplett leer sein!

Texteingabefelder:

  • Barcode: Nach Eingabe des Barcodes und Klicken auf Suchen werden "Wirkstoff" und "Bezeichnung" automatisch in die Texteingabefelder hinzugefügt.
  • Wirkstoff: Nach Eingabe des Wirkstoffes und Klicken auf Suchen werden "Barcode" und "Bezeichnung" automatisch in die Texteingabefelder hinzugefügt. (Sollte bereits Inhalt in "Barcode" stehen, wird der Begriff in "Wirkstoff" bei der Suche ignoriert.)
  • Bezeichnung: Nach Eingabe der Bezeichnung und Klicken auf Suchen werden "Barcode" und "Wirkstoff" automatisch in die Texteingabefelder hinzugefügt. (Sollte bereits Inhalt in "Barcode" oder "Wirkstoff" stehen, wird der Begriff in "Bezeichnung" bei der Suche ignoriert.)
  • Menge: Die Menge des Eintrages muss hinzugefügt werden.
  • Ablaufdatum: Das Ablaufdatum des Eintrages muss hinzugefügt werden. (Im Format MM/YY)

Suchen: "Barcode", "Wirkstoff" oder "Bezeichnung" werden automatisch in die Texteingabefelder hinzugefügt, wenn eines dieser 3 Textfelder ausgefüllt ist und, wenn der Eintrag bereits in der Tabelle vorhanden ist. Die Priorität der Suche ist Barcode, Wirkstoff, Bezeichnung. Sollte der Eintrag noch nicht in der Tabelle vorhanden sein, kommt die Meldung: Neues Produkt! In dem Fall müssen alle Texteingabefelder manuell ausgefüllt werden.

Neuer Eintrag: Ein neuer Eintrag mit den Begriffen/Werten der Texteingabefelder wird in die Tabelle hinzugefügt. Der Button funktioniert erst, wenn alle Texteingabefelder ausgefüllt sind.

Versionsverlauf: Der Versionsverlauf der Datei ist in der Cloud einsehbar. Der Account muss dafür freigegeben sein. https://www.dropbox.com/home/Apotheke

Sicherung der Einträge: Die Tabelle wird NUR alle 60 Sekunden automatisch auf die Cloud hochgeladen (Es erfolgt ein Hinweis). Daher wird Empfohlen, vor jedem Neuladen oder Schließen des Programm die Speicherung manuell durchzuführen (Speichern). Im Bearbeitungsmodus ist die automatische Speicherung unterbrochen.

Fehlermeldungen im Zusammenhang mit der Cloud-API: Es gibt 3 Arten von Fehlermeldungen im Zusammenhang mit der API. Generell gilt dabei: Wenn das Programm bereits eine Tabelle anzeigt, so wird diese automatisch extern gespeichert. Im Anschluss wird die Seite neu geladen. Ignorieren Sie ggf. die Standardmeldung des Browser, die vor Datenverlust warnt.

  • Access-Token fehlt: Authentifizierung ist nicht erfolgt.
  • Access-Token ungültig: Die Sitzung ist wahrscheinlich abgelaufen und muss erneuert werden.
  • sonstige Fehler: Alle anderen Fehler könnten Probleme mit der API sein. Bei Fragen: apotheke@leanderbrunar.net

Datensicherheit: Die Daten der Tabelle sind über eine sichere API auf einer Cloud (Dropbox) gespeichert. Der Zugang zur API ist nur nach erfolgreicher Authentifizierung möglich. Eine Authentifizierung ist nur möglich, wenn man den richtigen App-Key (= Passwort, das bei Sitzungsstart eingeben und verschlüsselt wird) hat.
